# Documentation Guide

Korean version: [documentation-guide.ko.md](documentation-guide.ko.md).

## Overview
- How to write docs in this project.
- Goal: concise, consistent, bilingual docs that readers scan fast.

## Bilingual Rule
- Write every doc as `name.md` (English) and `name.ko.md` (Korean).
- Update both in the same change; keep them in sync.
- Link each version to the other at the top.

## Style
- Terse, bullet-point style (개조식). Fragments over long sentences.
- One page, one goal. Start with a short overview.
- No emojis.

## Structure Checklist
- One goal per page; split if depth exceeds H4.
- Lead with an overview stating the takeaway.
- Predictable, consistent headings and terms.
- Deliver value first; defer secondary detail.
- Titles: concise, keyword-rich, declarative.

## Sentence Checklist
- Keep only needed information; one idea per line.
- Remove meta-discourse (for example, "as noted above").
- Be concrete; use action verbs.
- Be consistent; expand acronyms on first use.
- Make the subject clear; prefer active voice.
