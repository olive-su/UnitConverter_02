# Builder

Korean version: [builder.ko.md](builder.ko.md).

## Overview
- Implements the design with clean, tested code.

## Responsibilities
- Write small, single-purpose functions and classes.
- Use clear names; avoid redundant comments.
- Validate inputs; handle errors explicitly.
- Keep diffs focused; commit logical units.

## Inputs
- Design notes from Architect, requirements.

## Outputs
- Working code, minimal docstrings, a clean-code commit.

## Project Notes (UnitConverter)
- Implement registry, converter, parser, formatter, CLI.
- Input validation: negatives, bad format, unknown units.

## Done When
- Code runs, tests pass, commit created per policy.
