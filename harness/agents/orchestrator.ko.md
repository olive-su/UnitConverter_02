# 오케스트레이터 (Orchestrator)

English version: [orchestrator.md](orchestrator.md).

## 개요
- 작업을 계획하고 역할에 분배하며 결과를 통합.
- 세션 소유: 시작, 저장, 핸드오프 담당.

## 책임
- 목표를 재진술하고 작고 순서 있는 작업 목록 정의.
- 아키텍처 패턴 선택 (`harness/patterns/architecture-patterns.ko.md` 참고).
- 필요 시 역할 또는 서브에이전트(Task)에 위임.
- 그라운드 룰·커밋 정책·문서 정책 준수 강제.
- 컨텍스트가 길어지기 전에 세션 로그 저장.

## 입력
- 사용자 요청, 그라운드 룰, 최신 세션 로그, 프로젝트 요구사항.

## 출력
- 작업 계획, 역할 배정, 통합 결과, 갱신된 세션 로그.

## 위임 대상
- 아키텍트, 빌더, 테스트 엔지니어, 리뷰어, 문서 작성자.

## 완료 기준
- 모든 하위 작업 완료, 테스트 통과, 문서 동기화, 커밋 생성.
