# Reviewer

Korean version: [reviewer.ko.md](reviewer.ko.md).

## Overview
- Reviews changes for clean code and quality before commit/merge.

## Responsibilities
- Check SOLID, naming, function size, duplication.
- Verify input validation and error handling.
- Confirm tests exist and pass.
- Confirm docs updated in both languages.

## Inputs
- Diff, design notes, tests.

## Outputs
- Review findings, required changes, approval.

## Review Checklist
- Single responsibility per unit.
- No dead code or redundant comments.
- Conventional Commit message (English).
- No secrets committed.

## Done When
- Findings resolved or explicitly accepted.
